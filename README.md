## 需求分析报告
#### 一、课题内容分析
* 设计内容：设计一个齐齐哈尔大学校园咨询系统，为新生提供各种信息查询服务。
* 具体功能有：
 * (1)设计齐齐哈尔大学的校园平面图，以图中顶点表示校内各建筑，存放建筑名称、代号、简介等信息；以边表示路径，存放路径长度等相关信息。
 
 * (2)为来访客人提供图中任意建筑相关信息的查询。
 
 * (3)为来访客人提供图中任意建筑的问路查询，即查询任意两相建筑之间的一条最短的简单路径。
 
 * (4)提供图中任意建筑问路查询，即求任意两个建筑之间的所有路径。
 
 * (5)提供校园图中多个建筑的最佳访问路线查询，即求途经这多个建筑的最佳路径。
 
#### 二、设计思路
* 校园导游咨询系统模型是由建筑和建筑之间的路径组成的，所以可以用图的数据结构来实现。
* 用图的结点代表建筑，用图的边代表建筑之间的路径。
* 首先设计一个图类，结点值代表建筑信息，边的权值代表建筑间的距离。
* 结点值及边的权值使用文件存储，通过读取文件来获取建筑信息和建筑之间的距离。
* 计算任意两建筑之间的最短路径可以使用Floyd算法实现，使用深度优先遍历算法来实现两建筑之间的所有路径。
* 最后用switch选择语句来执行不同编号的功能。

#### 三、需求分析
##### 3.1 所需知识点

##### 3.2 数据结构
 * 本课题用到的数据结构是图的数据结构，其中用到的是图的邻接矩阵存储结构。
 
typedef struct {                             /*存放建筑信息的结构体*/
	int num;                                      /*建筑代号*/
	char name[20];                                /*建筑名称*/
	char intro[200];                              /*建筑简介*/
}vertextype;

typedef int edgtype;                            /*权值类型*/
typedef struct {                                /*校园建筑图结构体*/
	vertextype vexs[M];                           /*顶点信息域*/
	edgtype edge[M][M];                           /*邻接矩阵*/
	int vexNum, edgNum;                           /*图中顶点数和边数*/
}mgraphtype;
